这一次，你的任务是确定你需要替换的牌的最小数量，以使其成为赢牌。换句话说，就是要确定所谓的 "山田数"。请记住，不仅有标准的 "四组一对 "手型，还有 "千鸟 "和 "国士无双"。

为了简单起见，我们省略了开局的可能性。不过，您应该彻底检查输入数据，如果无效，则返回 "无"。

下面是解释性示例：

在下一手牌中，您只需要替换一张牌（"tempai"）：

['p4', 'p5', 'p6', 'm1', 'm1', 'm1', 's2', 's3', 's4', 's5', 'we', 'we', 'dw', 'dw']

而这一手有三块瓷砖需要替换：

['p4', 'p5', 'p6', 'm1', 'm1', 'm1', 's2', 's3', 's4', 's6', 'we', 'ws', 'dw', 'dw']

输入数据： 一个参数--编码瓦片的字符串列表。

输出数据： 一个 int 类型的变量：手牌中的山田数。

举例说明 

    riichi_mahjong_shanten(['p4', 'p5', 'p6', 'm1', 'm1', 'm1', 's2', 's3', 's4', 's5', 'we', 'we', 'dw', 'dw']) == 1
